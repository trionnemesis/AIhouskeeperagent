## DI-1 · PII 在 egress 遮罩（最高優先）

**規範**：實價登錄 2.0（2021/7 起）門牌/地號**完整揭露、來源未脫敏**（R1, verified）。PII 遮罩**不可**依賴來源，必須在 **AI Gateway PII 插件的 egress 出口**強制執行，且須在「**MCP post-tool 回傳 / 寫入 mem0 之前**」hook，**非僅** LLM I/O hook（否則資料已落 mem0）。

**遮罩對象**：門牌號、地號、自然人姓名、統編可反查之自然人、聯絡資訊。**法人統編可保留**（公開且業務需要）。

## DI-2 · 來源顯名

**規範**：政府開放資料（授權第1版 ≈ CC-BY）輸出時必標來源機關（如「資料來源：內政部不動產交易實價查詢服務網」）。未顯名視為自始未取得授權。OSM 資料須標 ODbL 署名。

## DI-3 · 資料時點揭露

**規範**：行情輸出附 `data_as_of`（資料更新至 YYYY-MM）+ **申報遲延警示**（近 1-2 季資料可能因申報遲延/解約而未完整）；禁稱「即時」。延續 [`../01-domain/invariants.md`](../01-domain/invariants.md) Inv-6。

## DI-4 · 來源黑名單

**規範**：`591`、`leju` 及任何「ToS 禁止 + 有技術保護措施」之來源，在 **ToolAuthZ 預設禁用**；放行條件＝法務意見書 + 授權證明（B2B 合作）。依據：公平會公處字第106084號裁罰先例、刑法§358/§359（繞過保護措施）、各站 ToS（R3, verified）。**robots.txt 不構成法律保護傘**；以 ToS/公平法見解為準。

## DI-5 · 治安粒度封頂

**規範**：治安/刑案資料**禁止**呈現為門牌或點位級精準（來源刻意降粒度到鄉鎮/分局/區域）；**禁止**把區域統計套到單一物件（=精準度造假，違零容忍）；**禁止**對物件貼「高犯罪/治安差」等負面標籤。A1/A2 交通事故雖有點位經緯度，呈現時仍以「周邊密度」聚合、附時間範圍。Policy 插件強制中性化。

## DI-6 · geocoding 信心門檻

**規範**：地址→座標若信心分數低於閾值，**不輸出精準座標**（改回「無法精準定位」或行政區級）。誤定位＝報錯位置；高敏感場景須人工覆核。

## DI-7 · provenance 必附

**規範**：每筆對外/入庫資料標 `{source, authority(政府|官方API|OSM|third_party_mcp|爬蟲), confidence, as_of}`；經 PII/Policy 過濾後才入 mem0；低信心或來源不可考者不入庫或標「不可對外輸出」。`third_party_mcp` 為通用類別，**本輪不消費任何第三方 hosted MCP（含 Twinkle Hub，見 [`twinkle-hub-alignment.md`](twinkle-hub-alignment.md)）**；自建 `tw-utils` 的輸出依底層官方資料標 `gov`/`official_api`。

## DI-8 · 配額 / ToS 遵守

**規範**：遵守各源 rate-limit（TDX 速率/收費、GCIS 每日次數/連線上限）；GCIS 落地散布須在「使用告知書」授權目的內（**個資法§20 特定目的外利用**風險）；自建 `tw-utils` 所用 open data（中華郵政/財金/TDX）授權須逐源確認（Q9）；ingest 逐 dataset 讀 license 欄位，**不可 hardcode 授權**。

## DI-9 · 行情快取時效

**規範**：行情類 cache **短 TTL**，過期重抓；plvr 每月 1/11/21 發布，揭露約 45 天前登錄之交易；**近 1-2 季因申報遲延/解約須定期重抓校正**。延續 Inv-5（過時即拒答/標警示）。
